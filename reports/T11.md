# T11 — Simplify schema editor for non-technical users

## Status: ✅ DONE

## Objective

Replace the dense 7-column grid field layout with a responsive stacked card layout that works on small screens and doesn't require JSON knowledge.

## Preflight

- Baseline: clean working tree after T10 commit `788bfcb`
- Build: `npm run build` exit 0, 63 routes

## Scope Confirmation

- **Target file:** `src/app/(app)/rulebooks/[id]/schema/page.tsx`
- **Before:** 7-column horizontal grid (`grid-cols-[auto_1fr_120px_80px_1fr_1fr_auto]`) — breaks on mobile, dense and intimidating
- **After:** Responsive stacked card per field with labeled inputs

## Execution Packet

### Changes Made

| Area | Before | After |
|------|--------|-------|
| Field layout | 7-column grid row | Card with 2-col grid (desktop) / 1-col (mobile) |
| Field inputs | Unlabeled inline inputs | Each input has a clear `<Label>` |
| Required toggle | Centered switch in grid column | Switch with "Required field" label below the form grid |
| Description | Inline input that expanded to textarea on focus | Simple input with "What this field represents" placeholder |
| Field identity | Drag handle (GripVertical) | "Field 1", "Field 2" numbered header |
| Imports | `GripVertical` imported | Removed unused import |

### Acceptance Criteria

| Criterion | Evidence |
|-----------|----------|
| Editor remains usable on smaller screens | `grid-cols-1 sm:grid-cols-2` responsive layout |
| User can add one field without needing JSON knowledge | Labeled inputs (Name, Type, Description, Example) with helpful placeholders |
