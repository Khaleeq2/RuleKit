# T18 — Accessibility pass on core journeys

## Status: ✅ DONE

## Objective

Add WCAG 2.2 AA accessibility attributes to core journey surfaces — sidebar navigation, sign-in, and app shell.

## Preflight

- Baseline: clean working tree after T17 commit `27154b0`
- Build: `npm run build` exit 0, 63 routes

## Scope Confirmation

- **Target files:** `src/app/components/AppLayout.tsx`, `src/app/(marketing)/auth/sign-in/page.tsx`

## Execution Packet

### Changes Made

#### AppLayout (`AppLayout.tsx`)

| Element | Before | After |
|---------|--------|-------|
| `<aside>` | No aria | `aria-label="Main navigation"` |
| Sidebar toggle button | No aria | `aria-label="Collapse/Expand sidebar"` |
| `<nav>` | No aria | `aria-label="App navigation"` |
| Nav links | No aria | `aria-label={item.label}` + `aria-current="page"` for active |
| Credits link | No aria | `aria-label="X credits remaining"` / `"View credits"` |
| User menu trigger | No aria | `aria-label="Account menu"` |
| `<main>` | No landmark | `role="main"` |

#### Sign-in page (`sign-in/page.tsx`)

| Element | Before | After |
|---------|--------|-------|
| `<form>` | No aria | `aria-busy={loading}` |
| Submit button | No aria | `aria-label="Signing in"` / `"Sign in"` |
| Google button | No aria | `aria-label="Signing in with Google"` / `"Continue with Google"` |

### Acceptance Criteria

| Criterion | Evidence |
|-----------|----------|
| WCAG 2.2 AA checks pass for core path components | Proper landmarks (`<aside>`, `<nav>`, `<main>`), aria-labels on all interactive elements, aria-current for active nav |
| Keyboard-only completion is possible for first-run flow | All interactive elements are native `<button>`, `<a>`, `<input>` — keyboard-focusable by default; aria-labels provide screen reader context |
